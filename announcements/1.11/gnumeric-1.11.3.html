Gnumeric 1.11.3 aka "TBD" is now available.

The Gnumeric Team is pleased to announce the availability of Gnumeric version 1.11.3. This release requires the concurrently released goffice 0.9.3.

Noteworthy news in this release:

  • Relicencing to GPLv2/GPLv3.
  • Import/export fixes for xls and ods.
  • Andreas
    • On ODF import of stacked column charts, set LibreOffice's default overlap. [Bug 670692].
    • On ODF import set series to default to ignoring invalids for interpolation. [Bug 671508].
    • Handle unusual styles created in xls import. [Bug 671745].
    • Do not show an icon for the range entry in the search dialog. [Bug 671494].
    • Import/Export style information attached to grids from/to ODF. Part of [Bug 671461].
    • Improve annotation import from ODF.
    • Fix conditional number format import from ODF. [Bug 672489].
    • Import/Export marker size from/to ODF. Part of [Bug 671461].
    • Export/Import axes data formats to ODF. Part of [Bug 671461].
    • Fix object anchors on ODF import. [Bug 667489].
    • Fix properties page of Document Properties dialog. [Bug 673129, Bug 673256].
  • Jean
    • Fixed crash with sheet object. [Bug 671617].
    • Fixed labels vectors made of multiple ranges. [Bug 672330].
    • Never skip a required plot dimension. [Bug 674341].
  • Morten
    • Hook up unknown_image icon for ssconvert too.
    • Fix xls save crash. [Bug 671267].
