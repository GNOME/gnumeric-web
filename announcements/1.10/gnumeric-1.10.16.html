Gnumeric 1.10.16 aka "TBD" is now available.

The Gnumeric Team is pleased to announce the availability of Gnumeric version 1.10.16.

Noteworthy news in this release:

  • A crash-on-start with some glib versions was fixed.
  • ODF and XLSX export improvements.
  • Andreas
    • Store some print range information in gnumeric files. [Bug 649714].
    • Hide Prefer CLIPBOARD preference under MS Windows. [Bug 649675].
    • Extend the goto dialog.
    • Extend the address box to handle both A1 and R1C1 style addresses. [Bug 650941].
    • Change the default widget when rolling up or down the default widget. [Bug 651173][Bug 646612].
    • Import external references from ODF. [Bug 645580].
    • Some documentation improvements.
    • Read fraction formats correctly from ODF 1.2 files.
    • Fix regression line export to ODF.
    • Fix font and background colour import of LibreOffice generated xlsx files.
    • In xlsx files, do not ignore gradient cell background fills completely. [Bug 644496].
    • Export some cell formatting to xlsx files. [Bug 532635].
    • Fix reading of diagonal borders from xlsx files. [Bug 652690].
    • Fix writing to xlsx of file with comments. [Bug 652689].
    • Add exporter for ECMA-376 2nd edition.
    • Fix some font-name handling in ODF import.
  • Morten
    • Fix leaks in SHEET. [Bug 650761].
    • Fix leaks in SHEETS. [Bug 650764].
    • Fix leak in name dialog.
    • Switch to using goffice's complex math.
    • Fix graph axis problem. [Part of Bug 599901].
    • Drop unneeded value formats. [Bug 651667].

goffice 0.8.16 aka "TBD" is now available.

  • Andreas
    • Improve ODF 1.2 output of fraction format.
  • Morten
    • Fix long-double compilation issue.
    • Improve accuracy of complex power.
    • Fix GOFormat problem resulting in "-0". [Bug 650928].
